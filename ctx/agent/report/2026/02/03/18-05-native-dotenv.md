# Отчёт итерации — native-dotenv

## Резюме изменений

Обновлён composition root backend: загрузка `.env` выполняется нативно и явно в `bin/app.mjs`, значения читаются только через `process.env`. Зависимость `env` удалена из `package.json` и `package-lock.json`.

## Детали работ

- Удалён импорт `env` и переход на `process.env` для выбора режима запуска в `bin/app.mjs`.
- Удалён пакет `env` из зависимостей в `package.json` и `package-lock.json`.
- Сохранена нативная асинхронная загрузка `.env` и правило неперезаписи существующих переменных окружения.

## Результаты

- Точка входа backend остаётся единственным местом работы с `process.env` и загрузкой `.env`.
- Код остаётся совместимым с ESM и инженерными конвенциями проекта.
- Зависимости для dotenv-функциональности отсутствуют.
