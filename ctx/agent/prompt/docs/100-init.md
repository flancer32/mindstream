# Mindstream — стартовый промпт разработки документации MVP

Мы работаем над проектом **Mindstream** — персональной смысловой лентой,
использующей LLM и эмбеддинги для фильтрации публикаций по смыслу и вниманию,
а не по тегам или времени.

Проект ведётся по методологии **ADSM (Agent-Driven Software Management)**.
Документация является **когнитивным контекстом**, определяющим границы и инварианты
разработки MVP. Код является следствием этой документации.

Цель текущей и последующих сессий — **последовательно создать корпус документации,
достаточный для разработки MVP одним человеком за короткий срок**, без расползания
смыслов, фич и архитектуры.

---

## Принципы работы

- Мы разрабатываем **только документацию**.
- В рамках одного диалога создаётся **один документ**.
- Документы декларативны: фиксируют _что истинно_, а не _как делать_.
- Каждый документ:
  - компактный;
  - плотный;
  - не дублирует другие уровни;
  - чётко обозначает свои границы.
- Фокус строго на **MVP**, не на продукт «в будущем».

Запрещено:

- предлагать новые фичи вне MVP;
- смешивать уровни (продукт / ограничения / архитектура / композиция / код);
- переходить к коду, если это не кодовый документ;
- описывать очевидное поведение стандартных технологий.

---

## План разработки документации MVP

Документы создаются **строго в следующем порядке**  
(от смысла к инженерии):

### Этап 1. Смысл продукта

1. `ctx/docs/product/vision.md` — зачем существует Mindstream
2. `ctx/docs/product/terminology.md` — базовые термины и язык проекта
3. `ctx/docs/product/AGENTS.md` — границы продуктового слоя

### Этап 2. Ограничения MVP

4. `ctx/docs/constraints/mvp-scope.md` — жёсткие рамки MVP
5. `ctx/docs/constraints/AGENTS.md` — приоритет ограничений

### Этап 3. Архитектурная форма

6. `ctx/docs/architecture/overview.md` — общая форма системы
7. `ctx/docs/architecture/data-flow.md` — потоки данных и внимания
8. `ctx/docs/architecture/AGENTS.md` — границы архитектурных решений

### Этап 4. Прикладная композиция

9. `ctx/docs/composition/ui.md` — минимальный пользовательский интерфейс
10. `ctx/docs/composition/AGENTS.md` — границы композиционного слоя

### Этап 5. Инженерия и среда

11. `ctx/docs/code/conventions.md` — минимальные инженерные нормы
12. `ctx/docs/code/AGENTS.md` — правила работы агента с кодом
13. `ctx/docs/environment/runtime.md` — среда исполнения и зависимости

Результатом выполнения плана является **полный когнитивный корпус**, достаточный
для разработки MVP Mindstream без добавления новых смысловых уровней.

---

## Процесс текущей сессии

1. Мы выбираем **один документ** из плана.
2. Ты помогаешь разработать его содержание в рамках ADSM.
3. Документ считается завершённым и будет подгружаться в контекст следующих сессий.
4. Следующая сессия продолжается со следующего документа по плану.

---

## Требования

- Не обрезай по длине абзацы в markdown-файлах при их генерации. В проекте используются современные IDE, они сами сворачивают код по ширине экрана.

## Начало диалога

Перед началом работы **запроси у меня название и путь документа**, который мы будем разрабатывать в рамках **текущего диалога**.
